##### 必填条件

| 条件项 | 内容 |
|---|---|
| 场景入口/触发方式 | 首页 > 商品详情页 > 立即购买 > 订单确认页 |
| 执行用户/角色 | 普通用户 user_a |
| 前置条件 | 用户已登录并具备下单权限；库存服务可用；测试商品、库存数量和收货地址可按用例准备 |
| 测试数据因子 | 商品状态：上架/下架；库存数量：0-9999；收货地址状态：有效/缺失；用户下单权限：有/无；订单状态：待支付 |
| 业务设计约束 | 库存充足、商品上架且收货地址有效时允许提交订单；库存不足、商品下架或收货地址缺失时阻止提交订单；提交成功后订单状态为待支付 |

##### 补充说明

关键操作包括点击“立即购买”，并在订单确认页点击“提交订单”。
提交成功后订单状态为待支付；提交失败时不得生成订单或扣减库存。
库存不足时提示“库存不足”；收货地址缺失时提示用户维护收货地址。
提交订单无二次确认弹窗。

|---|---|---|

| 性能 | 资源效率测试 | 验证系统完成功能或提供性能时，与所使用资源之间的比率是否达成设计预期，并确认是否满足目标网络场景的商用要求。 |
| 兼容性 | 配套兼容性测试 | 指对厂家内部两个或两个以上功能实体能否兼容运行、为客户提供既定功能和性能能力的测试。包括版本配套兼容性、网络兼容性、软件版本前向/后向兼容性、硬件一致性等，应包含但不限于软件版本差异、器件差异、单板差异、整机配合一致性。 |
| 兼容性 | 互通测试 | 不同厂商、不同类型设备或软件之间的协同工作测试。 |

| 可靠性 | 过载测试 | 通过使系统达到或超过其设计最大负荷，检验系统在非正常状态下的特性表现。这里的特性主要指系统所需完成的正常功能。 |
