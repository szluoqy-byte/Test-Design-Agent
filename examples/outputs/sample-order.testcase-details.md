# 订单提交测试用例明细

### TC-001 验证普通用户在有效订单数据下成功提交订单

- 用例编号：TC-001
- 用例名称：验证普通用户在有效订单数据下成功提交订单
- 用例等级：Level 0
- 前置步骤：
  1. 存在一个具备下单权限的普通用户账号。
  2. 系统存在一个库存大于 1 的可售商品。
  3. 该用户已配置一个有效收货地址。
- 测试步骤：
  1. 使用普通用户账号登录系统。
  2. 进入商品购买入口并选择库存充足的目标商品。
  3. 进入订单确认页并确认收货地址、商品数量和订单金额。
  4. 提交订单。
- 预期结果：
  1. 用户登录后系统展示已登录状态。
  2. 订单确认页展示的商品、数量、收货地址和金额与选择内容一致。
  3. 订单提交成功，系统生成订单号。
  4. 新订单状态为待支付。

### TC-002 验证普通用户在库存不足时提交订单被拦截

- 用例编号：TC-002
- 用例名称：验证普通用户在库存不足时提交订单被拦截
- 用例等级：Level 1
- 前置步骤：
  1. 存在一个具备下单权限的普通用户账号。
  2. 系统存在一个库存为 0 的商品。
  3. 该用户已配置一个有效收货地址。
- 测试步骤：
  1. 使用普通用户账号登录系统。
  2. 进入商品购买入口并选择库存为 0 的目标商品。
  3. 尝试提交订单。
- 预期结果：
  1. 系统阻止订单提交。
  2. 页面展示库存不足提示。
  3. 系统不生成新订单号。
  4. 商品库存状态保持为库存不足。
