# 覆盖与追溯标准

## 追溯矩阵字段

| 字段 | 要求 |
|---|---|
| 测试点 ID | 输入测试点编号，例如场景测试点 `TP-001` 或接口测试点 `ITP-001` |
| 测试点摘要 | 简短描述测试点目标 |
| 覆盖用例 | 一个或多个 `TC-*` 编号 |
| 覆盖状态 | `已覆盖`、`部分覆盖`、`待确认`、`未覆盖` |
| 备注 | 覆盖不足或待确认原因 |

## 覆盖状态

- `已覆盖`：至少一条可执行用例覆盖测试点主要目标。
- `部分覆盖`：只覆盖部分条件、路径、数据或预期。
- `待确认`：缺少必要业务信息，暂不能生成可靠用例。
- `未覆盖`：质量门禁发现遗漏，必须修正或说明原因。

## 规则

- 每个有效测试点至少被一条用例覆盖。
- 每条测试用例应能通过追溯矩阵关联至少一个测试点。
- 追溯矩阵是测试点与测试用例的唯一关联位置。
- 高风险测试点的覆盖应体现主路径、关键异常和必要边界。
- 一条用例可以覆盖多个测试点，但这些测试点必须属于同一测试场景，并共享同一执行链路、前置条件和主要观察点。
- 不应把不同条件、不同分支、不同异常、不同状态迁移或缺陷定位方向不同的测试点合并到一条用例中。
