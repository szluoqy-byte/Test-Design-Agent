# 接口契约测试设计

## 方法定义

接口契约测试设计用于验证系统对 API、消息、回调或第三方交互契约的遵守情况。契约包括请求字段、响应字段、错误码、鉴权、幂等、超时、重试和数据落库规则。

该模式可以指导接口测试，也可以指导前后台交互或消息驱动场景的测试用例设计。

## 适用场景

- HTTP API、RPC、消息队列、回调通知。
- 字段必填、类型、枚举、格式、长度、范围校验。
- 鉴权、签名、权限、租户隔离。
- 错误码、错误信息、响应结构。
- 幂等、重复请求、超时、重试、回调补偿。
- 第三方系统或外部依赖交互。

## 不适用场景

- 纯页面流程且无明确接口契约，应使用场景测试。
- 多条件业务规则矩阵，应使用判定表。
- 重点是稳定性恢复，应叠加可靠性与恢复性模式。

## 建模输入

| 元素 | 说明 |
|---|---|
| 接口/消息 | 名称、路径、主题或回调地址 |
| 请求契约 | 必填字段、类型、格式、枚举、范围 |
| 响应契约 | 状态码、业务码、字段、错误信息 |
| 鉴权契约 | token、签名、角色、租户 |
| 副作用 | 数据落库、状态变化、通知、日志 |
| 异常契约 | 超时、重试、幂等、依赖失败 |

## 设计步骤

1. 明确接口入口和调用主体。
2. 识别请求字段和响应字段。
3. 设计合法请求成功用例。
4. 设计字段级校验用例：必填、类型、格式、枚举、范围。
5. 设计鉴权和权限用例。
6. 设计错误码和错误信息用例。
7. 对高风险接口补充幂等、重复请求、超时、重试和回调用例。
8. 验证接口副作用：数据库、状态、消息、日志。

## 用例派生规则

| 契约类型 | 派生方式 |
|---|---|
| 成功契约 | 合法请求返回成功响应并产生正确副作用 |
| 字段契约 | 字段缺失、非法类型、非法格式、越界值 |
| 鉴权契约 | 未登录、token 失效、角色无权限、租户越权 |
| 错误契约 | 业务失败时返回指定错误码和错误信息 |
| 幂等契约 | 重复请求不产生重复副作用 |
| 回调契约 | 回调成功、失败重试、重复回调 |

## 数量控制

- 字段很多时，优先测试必填、高风险和会影响金额/权限/状态的字段。
- 同类字段校验可选择代表字段。
- `Level 0/1` 接口必须覆盖成功、关键失败、鉴权和副作用。
- 可靠性场景较多时，转交可靠性与恢复性模式补充。

## 前置步骤关注

- 准备调用凭证、角色、租户和业务数据。
- 准备接口依赖数据和外部系统状态。
- 明确如何观察副作用，例如后台记录、状态或日志。

## 测试步骤关注

- 明确请求方法、入口、关键字段和触发方式。
- 对非法请求只改变一个关键字段。
- 对幂等用例需要重复发送相同业务请求。
- 对回调用例需要模拟回调成功、失败或重复。

## 预期结果关注

- HTTP 状态或消息消费结果正确。
- 业务错误码和错误信息正确。
- 响应字段完整且类型正确。
- 副作用正确且可追踪。
- 无权限或越权请求被拦截。
- 重复请求不产生重复订单、重复扣款或重复状态变化。

## 常见误区

- 只检查响应成功，不检查数据副作用。
- 字段校验只测缺失，不测类型、范围和枚举。
- 忽略鉴权、租户和幂等。
- 把接口错误统一写成“返回失败”，缺少错误码和可判定结果。

## 设计方案片段

| 覆盖意图 | 用例标题建议 | 预期关注 |
|---|---|---|
| 成功契约 | 验证合法请求创建订单接口成功 | 返回成功业务码并生成订单 |
| 字段校验 | 验证缺少必填字段时接口拒绝请求 | 返回指定错误码且不落库 |
| 幂等 | 验证重复提交相同请求时接口保持幂等 | 只生成一条业务记录 |
