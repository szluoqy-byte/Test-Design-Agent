# 处理周期测试设计 PCT

## 方法定义

处理周期测试设计用于验证按时间、批次、轮次或周期重复发生的处理过程。它关注周期边界、周期内重复、跨周期延续、失败重试和补偿。

PCT 适合账期、结算、批处理、定时任务、重试轮次等场景。它与状态转换不同：PCT 关注处理过程在周期维度上的行为，STT 关注对象状态机本身。

## 适用场景

- 账期、结算周期、对账周期。
- 日终、月终、批处理、定时任务。
- 重试周期、补偿周期、轮询周期。
- 跨周期数据延续或归属。
- 周期内重复处理或幂等处理。

## 不适用场景

- 单个对象状态迁移，应使用 STT。
- 多条件决定处理动作，应使用 DTT。
- 单次流程主路径，应使用场景测试。

## 建模输入

| 元素 | 说明 |
|---|---|
| 周期单位 | 天、月、账期、批次、轮次 |
| 周期边界 | 开始、进行中、结束、跨周期 |
| 触发方式 | 定时、手动、事件、重试 |
| 处理对象 | 订单、账单、库存、统计、消息 |
| 失败处理 | 重试、补偿、跳过、告警 |
| 幂等要求 | 重复执行是否产生重复结果 |

## 设计步骤

1. 明确周期单位和周期边界。
2. 明确处理触发方式和处理对象范围。
3. 设计周期开始、周期中、周期结束、跨周期用例。
4. 设计重复执行和幂等用例。
5. 设计失败、重试、补偿和恢复用例。
6. 对账务、库存、统计等数据类场景补充一致性验证。
7. 对高风险周期任务补充日志、审计和告警验证。

## 用例派生规则

| 场景类型 | 派生方式 |
|---|---|
| 周期开始 | 验证新周期初始化、数据归属和起始处理 |
| 周期中 | 验证正常处理和累计结果 |
| 周期结束 | 验证结算、冻结、归档或收口动作 |
| 跨周期 | 验证数据从旧周期到新周期的归属和延续 |
| 重复处理 | 验证幂等和重复执行防护 |
| 失败补偿 | 验证失败后可重试、可补偿、可追踪 |

## 数量控制

- `Level 0` 周期任务至少覆盖开始、结束、跨周期、失败补偿。
- 普通周期任务覆盖正常周期和关键边界。
- 非核心周期任务可只覆盖正常处理和失败提示。
- 多个周期单位类似时，选择风险最高的周期代表。

## 前置步骤关注

- 准备位于不同周期边界的数据。
- 明确当前系统时间、账期或批次。
- 准备已处理、未处理、处理失败的数据。
- 如果需要模拟定时任务，应说明触发方式。

## 测试步骤关注

- 明确触发周期处理的动作。
- 对重复执行场景，应连续触发两次或多次。
- 对跨周期场景，应先构造旧周期数据，再切换到新周期。

## 预期结果关注

- 数据归属到正确周期。
- 处理次数正确，不重复、不遗漏。
- 失败记录、重试记录和补偿记录可追踪。
- 日志、审计、告警符合要求。
- 周期切换不破坏历史数据。

## 常见误区

- 只测周期中正常处理，不测周期边界。
- 忽略重复执行导致的重复扣减、重复结算、重复通知。
- 把对象状态变化当作周期测试，遗漏时间边界。
- 对失败补偿只看任务成功，不看数据最终一致。

## 设计方案片段

| 覆盖意图 | 用例标题建议 | 预期关注 |
|---|---|---|
| 周期开始 | 验证新账期开始时系统初始化结算数据 | 新账期数据创建且归属正确 |
| 跨周期 | 验证跨账期订单按规则归属结算周期 | 数据进入正确账期 |
