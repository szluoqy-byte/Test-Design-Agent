# 状态转换测试设计 STT

## 方法定义

状态转换测试设计用于验证对象在事件触发下从一个状态迁移到另一个状态的行为。它关注状态集合、事件、动作、合法迁移、非法迁移和状态保持。

STT 适合审批、订单、任务、账户、工单等具有生命周期的对象。

## 适用场景

- 对象存在明确状态：待提交、待审批、已通过、已拒绝、已取消。
- 操作是否允许取决于当前状态。
- 事件会触发状态变化、回退、超时或终止。
- 存在非法迁移或不可逆状态。
- 页面按钮、接口动作随状态变化。

## 不适用场景

- 只有单个校验点，应使用 DPT。
- 重点是账期、批次或时间周期，应使用 PCT。
- 多条件共同决定动作但不涉及状态机，应使用 DTT。

## 建模输入

| 元素 | 说明 |
|---|---|
| 状态集合 | 对象可能处于的状态 |
| 初始状态 | 用例开始前对象状态 |
| 事件/操作 | 触发状态变化的动作 |
| 目标状态 | 事件成功后的状态 |
| 非法迁移 | 不允许发生的状态变化 |
| 副作用 | 通知、日志、数据锁定、按钮变化 |

## 设计步骤

1. 列出状态集合和初始状态。
2. 画出合法迁移：当前状态 + 事件 -> 目标状态。
3. 标记非法迁移和不可逆迁移。
4. 设计主状态链路用例。
5. 设计关键非法迁移拦截用例。
6. 设计超时、回退、撤回、取消、重试等特殊迁移。
7. 验证状态变化后的可操作入口、按钮、接口和数据记录。

## 用例派生规则

| 迁移类型 | 派生方式 |
|---|---|
| 合法迁移 | 从源状态触发事件，验证目标状态 |
| 非法迁移 | 从不允许的源状态触发事件，验证拦截和状态保持 |
| 状态保持 | 操作失败或条件不满足时状态不变 |
| 回退/撤回 | 验证状态回到指定状态 |
| 超时迁移 | 验证定时或超时事件触发状态变化 |
| 不可逆迁移 | 验证终态不允许再次操作 |

## 数量控制

- `Level 0/1` 覆盖主状态链路和关键非法迁移。
- 状态很多时，优先覆盖高风险状态和终态。
- 相同类型的非法迁移可选择代表状态。
- 状态按钮或权限差异明显时，需要补充专项用例。

## 前置步骤关注

- 准备处于指定初始状态的业务对象。
- 明确触发事件所需角色和权限。
- 对超时迁移准备系统时间或任务触发条件。

## 测试步骤关注

- 先确认对象处于源状态。
- 触发一个明确事件。
- 操作后观察状态、入口、按钮和记录。

## 预期结果关注

- 目标状态正确。
- 非法迁移被拦截，源状态保持。
- 状态变化产生必要通知、日志或审计。
- 状态变化后可用操作集合正确。
- 终态不可逆时不允许再次修改。

## 常见误区

- 只测主链路，不测非法迁移。
- 只看状态字段，不看按钮、权限和副作用。
- 把周期边界当成状态迁移处理。
- 未准备明确初始状态，导致用例不可执行。

## 设计方案片段

| 覆盖意图 | 用例标题建议 | 预期关注 |
|---|---|---|
| 合法迁移 | 验证待审批单据审批通过后进入已通过状态 | 状态变为已通过并记录审批日志 |
| 非法迁移 | 验证已取消订单不允许再次支付 | 支付入口不可用且状态保持已取消 |
| 超时迁移 | 验证订单超时未支付后自动关闭 | 状态变为已关闭并记录关闭原因 |
