# 等级一致性门禁

## 通过条件

- 用例等级与等级定义、测试点等级、业务影响和 memory 规则一致。
- 等级只使用 `Level 0` 至 `Level 4`。
- 高风险测试点不被降级为低等级用例，除非报告说明依据。

## 等级定义

| 用例等级 | 定位 |
|---|---|
| Level 0 | 核心。每个转测试版本必验，失败意味着版本核心功能受损 |
| Level 1 | 关键。每个迭代建议验证，失败意味着特性关键功能点阻塞 |
| Level 2 | 重要。TR 点或对外发布版本完整验证，覆盖系统重要功能 |
| Level 3 | 一般。版本变更较大或全量验证时，按变更范围选择回归 |
| Level 4 | 生僻。低频应用场景或特殊预置条件，新特性首次验证后按需回归 |
