# 测试用例设计

本 skill 负责从测试点生成用例设计方案。粒度边界读取 `knowledge/test-scenario-point-case-boundary.md`，测试类型识别读取 `knowledge/basic-test-types.md`，详细设计模式读取 `knowledge/testcase-design-patterns/README.md` 和对应模式文件。

## 职责边界

- 输出“如何设计”的方案，不输出完整用例正文。
- 选择测试设计模式并说明覆盖意图。
- 为 `testcase-writing` 提供用例标题、等级、前置关注、步骤关注和预期关注。
- 不编写最终的前置步骤、测试步骤和预期结果。

## 输入

- 规范化测试点清单。
- `context-pack.md` 中与本次相关的项目事实。
- 用户补充的本次设计约束。

## 设计步骤

1. 按 `knowledge/test-scenario-point-case-boundary.md` 检查测试点是否写成测试场景或测试用例，必要时提出拆分、上收或待确认建议。
2. 识别每条测试点的测试类型和主要信号：流程、边界、组合、判定、周期、状态、接口、历史缺陷、风险或质量属性。
3. 按 `knowledge/testcase-design-patterns/README.md` 选择一个主模式，必要时选择一个辅助模式。
4. 确定覆盖意图：主路径、异常路径、边界、非法迁移、周期边界、接口契约、风险防护、性能效率或可靠性恢复。
5. 按测试点等级控制用例数量：
   - `Level 0`：覆盖主路径、关键异常和必要边界。
   - `Level 1`：覆盖主路径和主要异常。
   - `Level 2`：覆盖代表性正向或异常场景。
   - `Level 3`：轻量覆盖，避免重复。
   - `Level 4`：候选补充，除非用户或 memory 要求纳入。
6. 输出设计方案表，供 `testcase-writing` 编写用例。

## 输出格式

| 测试点 ID | 测试点摘要 | 主设计模式 | 辅助模式 | 覆盖意图 | 建议用例数 | 默认等级 | 待确认信息 |
|---|---|---|---|---|---|---|---|

## 约束

- 不把“正向/异常”作为独立设计模式；它们是每种模式下的路径属性。
- 不把权限、数据一致性等业务关注点单独当作模式；应映射到判定表、判定点、场景、接口契约、状态转换、处理周期或风险设计。
- 不为了凑数量生成重复用例。
